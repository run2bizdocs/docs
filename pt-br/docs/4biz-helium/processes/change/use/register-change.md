# Cadastrar uma mudança 

Para registrar uma alteração, é necessário registrar previamente um contrato, uma unidade que está vinculada ao contrato, um funcionário, um questionário, o Conselho Consultivo de Mudanças e um grupo vinculado ao contrato associado aos Solicitantes (funcionários) já está registrado neste grupo. Se for necessário criar um novo funcionário, é necessário incluí-lo no Grupo de solicitantes no qual está relacionado ao contrato.
Além disso, é necessário ter registrado o portfólio de mudanças.

Antes de começar
------------------

 Para efetivar o cadastro de uma mudança é preciso previamente cadastrar um contrato, uma unidade e que ela esteja vinculada ao contrato, um colaborador, um questionário, o Comitê Consultivo de Mudanças, ter um grupo vinculado ao contrato associado os Solicitantes (colaboradores) já cadastrados a este grupo. Caso seja preciso criar um novo colaborador, é necessário incluí-lo ao Grupo de Solicitante no qual esteja relacionado ao contrato.
Ademais, é fundamental o cadastro prévio do portfólio de mudanças.

Procedimento 
--------------

1.	Acesse o menu principal Processos> Gerenciamento de alterações> Alterar;
2. Clique em "Opções" e depois em "Registrar";
3. Preencha os campos disponíveis:

### Solicitante

Para identificar um candidato, preencha os campos:

| Campos | Descrição |
| --- | --- |
| Nome (\ *) | Nome completo do solicitante |
| Contato (\ *) | Identifique o contato do usuário |
| Email (\ *) | Email do usuário |
| Telefone | Número de telefone do usuário |
| Extension | O número do ramal do usuário |
| Unidade (\ *) | Unidade do usuário |
| Localização física | Localização física do usuário |
| Outras informações | Informações adicionais do usuário |

(*) Indicar campo obrigatório

### Mudança

Para identificar uma alteração, preencha os campos:

| Campos | Descrição |
| --- | --- |
| Título (\ *) | Título identificando a alteração |
| Descrição da mudança (\ *) | Descrição da mudança |
| Efeito de não implementar a alteração | Efetivo se essa alteração não for implementada |
| Contrato | Contrato vinculado à alteração |
| Importância | Nível de mudança na importância, temos as opções: ** Importante ** envolve uma quantidade significativa de preparação e trabalha com situações complexas ou importantes, ** Significativa ** Mudança significativa; Alterações significativas envolvem preparação e trabalho, avaliação, autorização e planejamento para a mudança.Exemplos: A compra e instalação de um novo servidor, A segmentação da rede, ** Pequena ** pode ser avaliada e autorizada fora da autoridade do CAB ( por exemplo, por um coordenador).
| Impacto | Impacto que a mudança gera, temos as opções: ** Baixa **, ** Média ** e ** Alta **. |
| Urgência | Urgência para fazer a alteração, temos as opções: ** Baixa **, ** Média ** e ** Alta ** |
| Grupo de Executores | O grupo responsável pela execução da alteração |
| Notificação | Configure notificações entre "Enviar email ao criar a alteração", "Enviar email após a conclusão da alteração", "Enviar um email para o Conselho Consultivo de Alterações", "Enviar email para outras ações da Alteração" |

(*) Indicar campo obrigatório

!!! Resumo "NOTA"
    
        Para [Criar portfólio de alterações] (/ pt-br / 4biz-helium / processos / change / configuration / change-portfolio.html)
        
### Análise ágil de risco

Na área de análise de risco, três perguntas serão feitas:

* Existe o risco de interromper serviços importantes? - com as opções "Sim" ou "Não"

* Existe um risco de perda financeira? - com as opções "Sim" ou "Não"

* Existe um risco de imagem? - com as opções "Sim" ou "Não"

então uma representação do risco de 0 a 100 será apresentada.

### Planejamento

Para planejar a mudança, use os recursos:

| Guias | Descrição |
| - | - |
| Atividades | Crie uma área de trabalho ou vincule uma existente |
Data para planejar a alteração: no campo "Data de início prevista", a data de início do planejamento deve ser indicada. No campo "Data final prevista", a data final do planejamento deve ser indicada. Depois de indicar a data inicial e a data final, você pode verificar os conflitos clicando em "Exibir conflitos". Aprovação: "Data de aceitação" deve ter uma data para ser aceita, "Votação" deve ter uma data para votação, "Data de conclusão" deve ter uma data para conclusão |
| Ações | Adicione uma ação para a mudança |
| Notificações do sistema | Configure uma notificação do sistema para informá-lo da alteração |
| Documentos | Vincular conhecimento a mudar |
| Anexos | Inserir anexos para alterar |
| Anotações | Adicione notações relevantes para alteração |

[Atividades da fase de planejamento de mudanças] (/ pt-br / 4biz-hélio / processos / mudança / uso / mudança-planejamento-atividades.html)

5. Também é necessário indicar as informações preliminares de "planejamento", "plano de reversão" e "revisão e encerramento".

### Plano de reversão

Para planejar a reversão de alterações, use os recursos:

| Guias | Descrição |
| - | - |
Atividades Crie um espaço de trabalho ou vincule um existente |
Ações | Adicione uma ação para a mudança |
| Notificações do sistema | Configure uma notificação do sistema para informá-lo do plano de reversão de alterações |
| Documentos | Vincular conhecimento ao plano de estorno |
| Anexos | Inserir anexos no plano de estorno |
| Anotações | Adicione anotações relevantes ao plano de estorno |

[Registrar plano de mudança de reversão] (/ pt-br / 4biz-helium / processos / change / use / change-reversion-plan.html)

### Revisão e fechamento

Para revisar e concluir a alteração, use os recursos:

| Guias | Descrição |
| - | - |
| Revisão | Informe ao Usuário final sobre análise de risco e impacto da mudança |
| Atividades | Crie uma área de trabalho ou vincule uma existente |
| Incidentes | Vincule um incidente para revisar |
| Problemas | Vincular um incidente a uma revisão |
| Lesons aprendidos | Vincular conhecimento ao plano de reversão |
| Encerramento | Identifique o encerramento de uma alteração |
| Anexos | Inserir anexos |
| Anotações | Adicionar notações relevantes |

5. Clique em "Opções" e depois em "Salvar";

## Relacionamentos

### Iten de Configuração de mudança

Para vincular um item de configuração, selecione Cis de Alterar> Pesquisar item de configuração e selecione o item de configuração que deseja vincular.

O sistema permite usar um item já configurado para acessar o mapa de item de configuração.

[Relacionar itens à alteração] (/ pt-br / 4biz-helium / processos / change / use / relate-information-to-change.html)

### Serviço da mudança

Para vincular um Serviço de Mudança, selecione Serviços da Mudança> Adicionar Serviço e selecione o item de configuração.

### Questionários

Não é obrigatório, mas algumas alterações podem vir junto com um questionário. Pode ser usado para fazer uma pesquisa (avaliação), questionários podem ser respondidos e as respostas serão vinculadas à mudança. O questionário deve ser criado com antecedência.

### Papéis e responsabilidades

Para definir as funções e responsabilidades da mudança, preencha os campos:

| Campos | Descrições |
| - | - |
| Empregado | Nome de usuário |
| Função | Função que o usuário executará |
| Responsabilidades | Selecione a responsabilidade a ser vinculada ao usuário |

### Votação RFC

A equipe de mudança pode visualizar a pontuação da votação para aprovação da mudança.

[Aprovar alteração] (/ pt-br / 4biz-hélio / processos / mudança / uso / mudança-aprovação.html)

### Lançamento

Para vincular um release a uma alteração, procure um existente em "Pesquisar título" ou crie um novo release usando a opção "Registrar"

Problema ###

Para vincular um problema a uma alteração, procure um existente em "Procurar título" ou crie uma nova versão usando a opção "Registrar"

### Base de conhecimento

Para vincular uma base de conhecimento a alterar, procure uma existente em "Procurar título" ou crie uma nova versão usando a opção "Registrar"

### Bilhete

Para vincular um ticket a ser alterado, procure um existente em "Procurar título" ou crie uma nova versão usando a opção "Registrar"
